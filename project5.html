Choose Improve

Usability Testing Project

Background

Improve uses research-backed learning from improv, psychology, and brain science to help individuals and teams improve their personal and professional wellness (Improve Studios, LLC, n.d.). Individuals and groups can benefit from Improve's services, which include online and in-person classes as well as asynchronous e-mail content for members.

Problem Statement

To improve usability and accessibility of the website and evaluate users' experience with the website.

Solution

Report was created which talked about key findings under each process along with a set of recommendations which included changing text placement, making navigation easy for important things, having a user login profile option where one can set wellness goals, etc .

Challenges
  1. 1. Finding the right group of audiences within the wellness space was a challenge for the survey study.
  2. 2. Considering wellness is a larger domain, it was necessary to narrow the scope of the evaluation for comparative analysis.
  3. 3. Due to time constraints, A/B testing was not conducted on the website, which would have provided a more thorough analysis.

Development Process

  • 1

    Interviews

    In order to understand different user's journey with the product, one-on-one interviews were conducted with six users and a stakeholder meeting with the client. The takeaways collected from interviews, were used for qualitative theming, affinity wall analysis, persona, and scenario development.

  • 2

    Comparative Analysis

    The objective of this study was to identify and analyze the direct, parallel, partial, and indirect competitors of Improve. Seven comparative products/services from the wellness/comedy/educational technology markets were chosen for this study. We compared competitors' features as well as features that Improve lacks but competitors have.

  • 3

    Survey

    To gain more insights and discover what the average user wants out of a service like Improve, we constructed a survey which was distributed to a broad target population. For this study we did not have a strict definition for our target audience as the services offered by the website can be utilized by all age groups.

  • 4

    Heuristic Evaluation

    We identified relevant heuristics and created criteria for evaluating Improve's usability based on Nielsen's 10 Usability Heuristics for User Interface Design and WebCriteria(2002). Evaluation of the entire website was carried using nine heuristics and under each heuristic, addressed issues were rated on a severity scale of 3 or higher.

  • 5

    Usability Testing

    The type of Usability test adapted for Improve is Summative Testing. Since our client's product is already developed, we created test cases to validate the product against defined benchmarks. The focus was on users who have never used or have limited interaction with the Improve website. Understanding a novice user's perspective was one way to know if the functionalities present on the website are accessible or not.
